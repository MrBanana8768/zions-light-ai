# Compactor venv requirements (V2.0 Phase 3+ · pins audited V3.0, 2026-06-30).
#
# As of Phase 3 the compactor runs in its OWN venv (/opt/compactor-venv),
# fully separate from vLLM. That means these deps can resolve freely
# WITHOUT risk of disturbing vLLM's torch/transformers pins — the
# dependency-coupling bug class that caused the 1.9.x fire drills is now
# structurally impossible.
#
# Deliberately torch-free: fastembed uses ONNX runtime, and transformers
# is used tokenizer-only (AutoTokenizer works without a torch backend for
# fast/rust tokenizers). Keeps the compactor venv lean.
#
# Versions pinned EXACTLY for reproducible builds (V3.0). Re-audit against
# PyPI/OSV before each release and bump deliberately. transformers moved
# 4.x -> 5.x here: the last 4.x (4.57.6) carries open CVEs fixed only in
# 5.x, and we use it tokenizer-only on a known model, so 5.12.1 is both the
# clean and the correct choice — but the 4->5 major is the thing the rebuild
# boot self-test must confirm (fallback: transformers==4.57.6).

# --- proxy / API ---

transformers==5.12.1
